## EXECUTIVE SUMMARY

**The Opportunity:**
TaxGPT (competitor) has 5,510 keywords in their profile. Their `/answer/` pages (2,788 keywords) generate 6,422 traffic — their entire content engine. But from KD ≤ 15 keywords alone, there are **679 keyword targets** with a combined volume of **500,000+** that they are either ranking poorly for or not fully capitalizing on.

**The Play:**
Build a programmatic Q&A content network on supatax.ai that targets 150–200 of these high-volume, low-KD tax questions. At an average of 25–35 organic visits/page (conservative), 200 pages = 5,000–7,000 monthly traffic. Several individual pages can deliver 300–800 traffic on their own.

**Why this works in 60–90 days:**
- supatax.ai already has domain presence, tools, and impressions — not starting from zero
- These are KD 0–15 keywords — low competition, fast ranking potential
- Q&A format pages get featured snippets and AI Overviews fast (this is proven by TaxGPT)
- The domain name is an EMD advantage (tax + AI = source context is obvious to Google)

## PART 2: SITE ARCHITECTURE

### URL Structure
```
supatax.ai/                          → Landing page (commercial, AI tax tools)
supatax.ai/tools/                    → Free tools hub
supatax.ai/tools/[tool-name]/        → Individual free tools
supatax.ai/tax-answers/              → Q&A hub (links to all answer clusters)
supatax.ai/tax-answers/[slug]/       → Individual Q&A pages (primary traffic engine)
supatax.ai/guides/                   → Long-form pillar guides
supatax.ai/guides/[slug]/            → Individual pillar guides
```

### Why /tax-answers/ NOT /answer/ (like TaxGPT)
- "tax-answers" contains more semantic signal than "answer"
- It differentiates your source context from TaxGPT
- Better EMD reinforcement for supatax.ai as a tax knowledge source

### Internal Link Architecture (Koray's I-Node system)
**Homepage links to:**
- 3 most important pillar Q&A pages (directly, in hero section)
- Tools hub
- 3–5 cluster hub pages within body content

**Cluster hub pages link to:**
- All Q&A pages within that cluster (I-nodes in main content)
- Related cluster hub pages (contextual bridges)

**Individual Q&A pages link to:**
- 2–3 related Q&A pages (I-nodes, in annotation text)
- The relevant cluster hub page (1 link back up)
- The most relevant free tool (1 link)

**Rules:**
- Header: homepage link ONLY
- Footer: homepage link ONLY
- Max 3 contextual I-node links per Q&A page
- Anchor text = near-exact match of destination page's target keyword

## PART 8: TECHNICAL SEO REQUIREMENTS

### Site Speed (Koray: "Page speed is a health and trust issue")
- Target LCP < 2.5s
- Target CLS < 0.1
- Use static HTML for Q&A pages (fastest possible)
- Lazy load images below fold
- No JavaScript-dependent content rendering for above-fold text

### Crawl Efficiency
- Clean XML sitemap, submitted to GSC
- Robots.txt: block /admin/, /api/, nothing else
- No paginated archives — cluster hub pages serve as the navigation layer
- Canonical tags on all pages (self-referencing)

### Schema Priority
1. FAQPage on all Q&A pages (triggers featured snippets + AI Overview)
2. Article on all Q&A and guide pages
3. WebApplication on homepage only (for the AI tax assistant)
4. Organization on homepage
5. BreadcrumbList on all pages

### Header Structure
```
H1: [Target keyword — once, exactly]
H2: [Sub-intent — directly implies H1]
H3: [Specific detail or example]
```
Never skip heading levels. Never use H2 for decorative purposes.
