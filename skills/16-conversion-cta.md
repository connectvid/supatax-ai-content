# Skill 13: Conversion & CTA Strategy

## The Core Lesson

[WYOMING LESSON] A real user spent 2+ hours on the site, asked 25 WhatsApp questions, caught a pricing discrepancy ($60 vs $100 state fee), and didn't buy. The site was built for reading, not buying. This is the most expensive mistake you can make.

**Design the conversion funnel BEFORE writing content. Always.**

---

## The 3 Customer Failure Patterns

These are documented real failures from wyomingllc.co:

**Customer 1 (JPCheng, China):**
- 2+ hours on site, 25 WhatsApp questions
- Noticed $60 state fee listed as $100 in one place
- Lost trust instantly → did not buy
- Fix: Zero pricing inconsistencies. Show exact breakdown everywhere.

**Customer 2 (Kendal, UK Company):**
- High-value B2B enquiry
- Asked for email contact, got confused by form
- Dropped off → lost $1,000+ potential
- Fix: Email contact prominently available. No friction in contact options.

**Customer 3 (Kosovo):**
- Dropped off when document requirements seemed complex
- Fear of complexity killed the sale
- Fix: Simplify the "what you need to do" section. We handle the complexity.

## Non-Negotiable CTA Rules

### Every Page Must Have:

1. **Above-fold CTA** — visible without scrolling, on both mobile AND desktop
2. **Sticky CTA** — follows scroll on all content pages (WhatsApp green button)
3. **Transparent pricing** — always show exact breakdown (never just a total)
4. **WhatsApp within 1 tap** on mobile — no email forms that require typing
5. **"What's Included" visual** on every page that mentions a price
6. **Trust signals** near every CTA: reviews, guarantee, credentials

### CTA Text Variations (Use These, Don't Invent)
- "Apply for your ITIN today →"
- "Get your ITIN in 2-3 days with Express →"
- "Start your ITIN application →"
- "Get your ITIN now →"

### CTA Placement
- Position 1: Hero section (above fold, primary button)
- Position 2: After the Quick Answer section (users who skimmed and are ready)
- Position 3: After pricing section (capture decision moment)
- Position 4: After FAQ section (capture users after objections resolved)
- Position 5: Final CTA before footer (last chance)
- Sticky: WhatsApp button floating bottom-right on all pages

## Pricing Transparency Template

Never show just a total. Always show the breakdown:

```
Standard ITIN Service — $147
├── W-7 form preparation: included
├── Document review by CAA: included
├── Application submission to IRS: included
├── Email status updates: included
└── IRS processing (6-11 weeks): IRS handles this (free)

NOT included:
├── IRS processing time (this is IRS timeline, not ours)
└── Additional state fees (if applicable)

Express ITIN Service — $247
├── Everything in Standard: included
├── Priority queue (2-3 business day preparation): included
├── FedEx tracking: included
├── WhatsApp support: included
└── 100% money-back guarantee if IRS declines: included
```

**Rule: Never have conflicting numbers anywhere on site.** One wrong number (even $40 off) destroys trust permanently. Audit all pricing mentions monthly.

## The Conversion Funnel Journey

Map this BEFORE building any page:

```
1. AWARENESS: Organic search → landing on informational content
   Goal: Educate. Build trust. Get them to the service page.
   CTA: "Learn how our service works →"

2. CONSIDERATION: Service page → Comparing options
   Goal: Show why we're better. Price vs competitor. Speed vs competitor.
   CTA: "See our packages →"

3. DECISION: Pricing page → Choosing a package
   Goal: Remove objections. Show transparency. Show guarantee.
   CTA: "Apply now — $147 →"

4. ACTION: Checkout → Payment → Confirmation
   Goal: Frictionless. Stripe checkout. Clear next steps email.
   CTA: Stripe payment link

5. POST-PURCHASE: Confirmation → WhatsApp onboarding
   Goal: Set expectations. Build trust. Get review.
   Action: Automated WhatsApp message with timeline
```

## Mobile Conversion Requirements

Mobile users = majority of traffic. Test on real device, not just browser resize.

Checklist:
- [ ] Primary CTA visible without scrolling on iPhone 12 viewport
- [ ] WhatsApp button tappable with thumb (not too small, not in corner)
- [ ] Phone number tappable (tel: link)
- [ ] Pricing table doesn't require horizontal scroll
- [ ] Form fields have appropriate input types (email, tel, number)
- [ ] "Buy now" button is minimum 44×44px tap target

---

## Trust Signal Requirements

Every service page must show:
- IRS verification link (for ITIN service — links to IRS acceptance agent list)
- Client count ("500+ clients served" or real number)
- Trustpilot/Google rating with star visual
- Money-back guarantee (if offered) — stated clearly, not buried
- Processing time commitment — stated as "We submit within X days"
- "What's NOT included" — proactively state this to build trust

## Customer Question → Content Pipeline

Every question asked on WhatsApp/email:

```
Step 1: Log question in FAQ spreadsheet
Step 2: Check if it has search volume (DataForSEO quick check)
Step 3a: If volume > 200/month → Create standalone page
Step 3b: If volume < 200/month → Add as H2 section to most relevant page
Step 3c: If it's a trust/objection question → Add to service page FAQ
Step 4: Update within 48 hours
```

The JPCheng questions that should have been pages:
- "Is tax filing included?" → /what-is-included/ or service page section
- "What happens if my application is rejected?" → /itin-application-rejected/
- "Can I get a refund?" → Service page FAQ
- "How do I know you're legitimate?" → /about/ with IRS verification link
